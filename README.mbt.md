# mbt-embed

Tiny MoonBit-native CLI that recursively walks a folder and emits a single
`.mbt` file exposing every file's bytes through a `Map[String, Bytes]`.

## Layout

```
src/mbt-embed/       the CLI tool (native backend)
example/assets/      sample files to embed
example/consume/     example package that reads embedded bytes
```

## Why a tool, not `include_bytes!`?

MoonBit has no file-include macro yet, so the tool reads files at *build time*
and emits a `.mbt` with the bytes baked in as `Bytes::from_array([...])` literals.

## Build the generator

The generator uses `moonbitlang/x/fs`, which only provides native/js stubs, so
the generator itself is built on the **native** backend:

```
moon build --target native src/mbt-embed
```

## Run it

```
moon run --target native src/mbt-embed -- <input_dir> <output_file> [--package <name>]
```

* `<input_dir>`  — folder to embed recursively (key = path relative to it).
* `<output_file>` — path the generated `.mbt` source is written to.
* `--package <name>` — emitted as a comment only; MoonBit does not need a
  per-file `package` directive, but it documents where the file belongs.

Empty files are supported (the entry is `Bytes::new(0)`).

## Wiring it into a build

`moon` has no `pre-build` script hook, so do one of the following:

* run `moon run` with the args above from a wrapper shell script / Makefile
  target before `moon build`, or
* run the generator once and check in (or gitignore) the generated file.

## Generated file shape

```moonbit
// Generated by mbt-embed. DO NOT EDIT.
pub let entries : Map[String, Bytes] = Map::from_array([
  ("hello.txt", _embed_0()),
  ("sub/note.txt", _embed_1()),
])

///|
fn _embed_0() -> Bytes { Bytes::from_array([0x68, 0x69, ...]) }

///|
pub fn get(path : String) -> Bytes? { entries.get(path) }
```
